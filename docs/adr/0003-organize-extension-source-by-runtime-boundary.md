# Organize Extension Source By Runtime Boundary

Bridge organizes TypeScript source by Chrome extension runtime boundary before feature area: `src/extension/service-worker/` for MV3 event orchestration, `src/extension/sidepanel/` for the extension UI page, `src/extension/content/` for code that runs against the original page DOM, `src/extension/platform/` for Chrome API wrappers, `src/domain/` for Chrome-independent Guided Task Mode rules, `src/providers/` for plan-creation adapters, and `src/backend/` for the local backend proxy. This makes the service worker's no-DOM, event-driven lifecycle explicit, keeps page-DOM code out of the service worker, and gives future refactors a safe path for shrinking `service-worker/main.ts` without changing the Guide-Only Assistance product boundary.
