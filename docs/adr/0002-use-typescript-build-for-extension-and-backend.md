# Use TypeScript Build For Extension And Backend

Bridge will keep its current Guided Task Mode behavior and public runtime contracts while moving the extension and backend source into TypeScript. The repository root remains the unpacked Chrome extension folder, but `manifest.json` points Chrome at compiled files under `dist/` instead of root source files. Bridge will use a small Node build stack with TypeScript for type checking and esbuild for bundling extension and backend entry points; `dist/` remains generated output and is not committed. After compiled equivalents exist, the legacy root runtime files are removed so `src/` is the single source of truth, organized around MV3 runtime boundaries such as `src/extension/service-worker/`, `src/extension/sidepanel/`, `src/extension/content/`, `src/domain/`, `src/providers/`, and `src/backend/`. The backend remains a dependency-light Node HTTP server, not an Express-style framework migration. This adds a build step while keeping the current load path stable and gives the guide session lifecycle, Plan Contract, provider messages, storage keys, and privacy boundaries shared typed contracts across the MV3 service worker, side panel, injected page scripts, and backend proxy.
